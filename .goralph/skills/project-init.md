# Project Initialization Skill - Generic

## Objective
Set up the project with a clean, maintainable structure.

## Steps

### 1. Initialize Version Control
```bash
git init
echo "# Project Name" > README.md
```

### 2. Create Basic Structure
Organize code into logical directories:
- Source code in dedicated folder
- Tests separate from source
- Configuration at root
- Documentation in docs/

### 3. Set Up Development Environment
- Create dependency manifest
- Add .gitignore for your stack
- Set up linting/formatting

### 4. Create README
Document:
- Project purpose
- Setup instructions
- Usage examples
- Contributing guidelines

## Completion Criteria
- [ ] Project builds/runs
- [ ] Structure is clean and logical
- [ ] README provides clear guidance
