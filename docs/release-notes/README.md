# Release Notes

Version release documentation. Record what shipped in each version.

## File Format

**Standard filename**: `cosmo-smoke-v{version}-ReleaseNotes.md`

Examples:
- `MyProject-v0.1.0-ReleaseNotes.md` - Initial release
- `MyProject-v1.0.0-ReleaseNotes.md` - First stable
- `MyProject-v1.2.3-ReleaseNotes.md` - Patch release

**Format rules**:
- Project name from `package.json` name or directory name
- Version with `v` prefix
- Suffix always `-ReleaseNotes.md`

## Contents

Categorize by: Added, Changed, Fixed, Removed
- New features and capabilities
- Bug fixes and improvements
- Breaking changes and migration guides (highlight prominently)
- Known issues
