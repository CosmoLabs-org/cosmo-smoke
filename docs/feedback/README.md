# Feedback & Improvement Proposals

Feature requests, improvement proposals, and tool feedback for upstream dependencies.

## File Naming

- `{tool}-{feature}-{version}.md` - Detailed proposals
- `HANDOFF-{topic}.md` - Ready-to-paste prompts for other projects

## Writing Good Feedback

1. **Context**: What were you trying to do?
2. **Problem**: What didn't work or was missing?
3. **Proposal**: What should be built?
4. **Priority**: How urgent is this?
5. **Acceptance Criteria**: How do we know it's done?
