# Brainstorming — How to Use

Store brainstorming sessions, design docs, and exploratory thinking.

## Commands

| Command | What it does |
|---------|--------------|
| `/brainstorm "topic"` | Start a brainstorming session |
| `/brainplan "topic"` | Brainstorm → plan → prompt pipeline |

## Workflow

1. Capture exploratory thinking in dated markdown files
2. Use `/brainstorm` for open-ended exploration
3. Use `/brainplan` when exploration should lead to implementation
4. Promote actionable ideas to issues or roadmap items

## Conventions

- Files named `YYYY-MM-DD-topic.md`
- Can contain design docs, architecture decisions, tradeoff analysis
- If content is detailed enough, use directly as implementation input
