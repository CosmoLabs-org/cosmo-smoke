# cosmo-smoke - SOP Brainstorming

Ideas and rough drafts for project-specific SOPs before implementation.

## Workflow

```
docs/brainstorming/SOPs/    →  Ideas, rough analysis (THIS FOLDER)
         ↓
docs/sops/                  →  Implemented project SOPs
```

## Adding Ideas

When you identify a repeated workflow or process:

1. Create a brainstorm file: `YYYY-MM-DD-sop-idea-name.md`
2. Document:
   - What problem it solves
   - Steps involved
   - Tools/skills needed
   - Pain points
3. When ready, implement in `docs/sops/`

## Template for Brainstorm

```markdown
# SOP Idea: [Name]

**Date**: YYYY-MM-DD
**Status**: Idea | Exploring | Ready to Implement

## Problem
What workflow is this trying to standardize?

## Current Pain Points
- Issue 1
- Issue 2

## Proposed Solution
How would an SOP help?

## Steps Identified
1. Step 1
2. Step 2

## Tools/Skills Needed
- Tool 1
- Skill 1

## Open Questions
- Question 1?
```
