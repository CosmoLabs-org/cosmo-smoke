# Session: v0.2.0 — Feature Expansion

**Date**: 2026-04-16
**Version**: v0.2.0
**Focus**: Core assertion expansion and testing toolkit transformation

## Summary

Transformed cosmo-smoke from a basic smoke tester into a comprehensive testing toolkit. Added HTTP endpoint assertions, JSON field assertions, config inheritance, and expanded from 5 to 10 assertion types. Pushed to GitHub as CosmoLabs-org/cosmo-smoke.

## Key Changes

- Add HTTP endpoint assertion with status/body/header checks
- Add JSON field assertion using gjson for JSONPath queries
- Add config inheritance via `includes:` directive and Go templates (`{{ .Env.FOO }}`)
- Add TAP v14 and JUnit XML output formats (4 total)
- Add `--from-running` container inspection for automatic test generation
- Add `smoke serve` HTTP health endpoint for Docker/K8s probes
- Add port_listening, stderr_matches, env_exists assertion types
- Create 7 example configs (go-api, node-fullstack, python-fastapi, docker-compose, rust-cli, monorepo, kubernetes)
- Dispatch 4 GLM agents (3 timed out, recovery patches saved)
- Capture 13 ideas + 21 roadmap items
