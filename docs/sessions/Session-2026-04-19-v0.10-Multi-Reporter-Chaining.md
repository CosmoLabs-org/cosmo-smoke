## Summary

This session picked up from a continuation prompt expecting a slate of roadmap items, but all of them had been completed in prior sessions. Rather than idle, we brainstormed and shipped a new feature: multi-format reporter chaining. The idea is straightforward -- `smoke run --format terminal,json` should write terminal output to stdout and a JSON report to a file, simultaneously -- but the existing code had three separate switch blocks in `cmd/run.go` that each mapped format strings to reporter constructors, making it clumsy to compose them.

We wrote a design spec first (`docs/brainstorming/2026-04-19-multi-reporter-chaining.md`) that laid out the Chain() factory pattern and a unified format registry. The registry was the key architectural decision: instead of maintaining one map for terminal reporters, another for JSON reporters, and a third for push reporters, a single `map[string]formatEntry` holds the constructor, output destination (stdout/file), and whether the format supports push reporting. This eliminated the three duplicated switch blocks in `cmd/run.go` and replaced them with a `buildReporter` helper that splits the comma-separated format string, looks up each entry in the registry, and assembles a `MultiReporter`. The refactoring caught a bug along the way: the non-monorepo code path was missing the `withPushReport` call, meaning push reports only worked under `--monorepo`. That's now fixed.

On the test front, 14 new tests cover the Chain() factory (empty input, single format, multiple formats, unknown format handling, format ordering, duplicate detection, push-only exclusion) plus the format registry (lookup, iteration, existence checks). All 378 tests pass, build is clean, and CLAUDE.md has been updated to document the comma-separated format syntax. One known limitation carried over from before: watch mode combined with file-based reporters can accumulate state across re-runs, but that predates this feature.

## Key Decisions

| Decision | Options Considered | Why This Choice |
|----------|-------------------|-----------------|
| Unified format registry over separate maps | (A) Keep 3 separate switch blocks, (B) Merge into single registry map | Single source of truth eliminates the bug class where one switch gets updated but another doesn't (the missing `withPushReport` was exactly this pattern) |
| Comma-separated `--format` flag | (A) Repeated `--format` flags, (B) Comma-separated string | Comma-separated is backward-compatible (single format strings still work) and avoids CLI flag accumulation complexity |
| Chain() as a pure factory function | (A) Method on existing Reporter interface, (B) Standalone factory | Pure function is testable in isolation, doesn't pollute the Reporter interface, and keeps the chain assembly logic in one place |
