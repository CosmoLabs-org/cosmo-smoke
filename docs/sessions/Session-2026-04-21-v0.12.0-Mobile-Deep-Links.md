# Session: Mobile Deep Link Assertion + Detector Templates

**Date**: 2026-04-21
**Version**: v0.12.0
**Focus**: Mobile deep link assertion type, mobile project detection templates

## Summary

Added a two-tier progressive deep link assertion (`deep_link`) for Android, iOS, React Native, and Flutter projects, with tier 1 HTTP config checks and tier 2 tool-augmented resolution. Introduced four mobile project types (React Native, Flutter, iOS, Android) into the detector with corresponding `smoke init` templates. Fixed watch mode config reload and TraceHealth persistence across runs.

## Key Changes

- **Deep link assertion** (FEAT-013): New `deep_link` assertion with two tiers — HTTP-based config verification and optional tool-augmented resolution for Android assetlinks, iOS AASA, and universal links
- **Mobile project detection**: Added React Native, Flutter, iOS, and Android project types to the detector (`internal/detector/`)
- **Smoke init templates**: Mobile project templates auto-generate `.smoke.yaml` configs tailored to each mobile framework
- **DeepLinkCheck struct**: New schema struct with fields for `url`, `android_package`, `ios_bundle_id`, `ios_associated_domains`, `check_assetlinks`, `check_aasa`, and `tier`
- **Pipeline wiring**: Wired `deep_link` assertion into the test execution pipeline (`internal/runner/`)
- **Watch mode fix**: Fixed config reload and TraceHealth persistence across runs in watch mode
- **Test expansion**: 107 commits, 292 files changed — broad test coverage additions across reporter, runner, schema, cmd, detector, and dashboard packages via parallel GLM agent batch
