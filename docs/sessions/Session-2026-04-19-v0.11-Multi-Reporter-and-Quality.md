# Session: Multi-Reporter Chaining + Quality Hardening

**Date**: 2026-04-19 to 2026-04-20
**Versions**: v0.11.0, v0.11.1
**Focus**: Multi-format reporter chaining, JUnit enhancements, validate/schema commands, baseline tracking

## Summary

v0.11.0 introduced multi-format reporter chaining, allowing `--format terminal,json,prometheus` to output to stdout and auto-named files in a single run. v0.11.1 followed with four new features (`smoke validate`, `smoke schema`, baseline tracking, JUnit CI metadata) and a bug fix for watch mode reporter state accumulation. Together the two releases comprised 17 commits across 62 files.

## Key Changes

- **Multi-format reporter chaining** (v0.11.0) — comma-separated `--format` flag; first format to stdout, rest to auto-named files (`smoke-results.json`, `smoke-metrics.prom`)
- **`smoke validate`** (v0.11.1) — standalone config validation without running tests
- **`smoke schema`** (v0.11.1) — export all assertion types as JSON for tooling integration
- **Performance baseline tracking** (v0.11.1) — `--baseline` and `--baseline-threshold` flags for regression detection
- **JUnit XML enhancements** (v0.11.1) — timestamp, hostname, and properties metadata for CI dashboards
- **Watch mode state reset fix** (v0.11.1, BUG-001) — reporters recreated per watch cycle to prevent unbounded state accumulation
