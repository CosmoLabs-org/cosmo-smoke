## Workflow Pattern Validated

This session proved a repeatable parallel-agent execution model:

1. **Opus decomposes** work into bounded tasks with exact briefs — file paths, function
   signatures, and expected output specified before dispatch.
2. **Sonnet agents execute** in isolated git worktrees with conventional commits. No
   cross-agent dependencies; each worktree is self-contained.
3. **Opus reviews** via `ccs verify-worktree` quality gate before any merge.
4. **If score < 7/10, Opus fixes in-worktree** before merge rather than merging and
   patching. This session: ROAD-005 (process_running) was hardened — pgrep -f replaced with
   -x, timeout added, lookup errors distinguished from "not running" errors.
5. **Sequential merges** with union-resolve strategy for adjacent-line conflicts in shared
   files (runner/schema).

7 agents total: 2 initial (process_running, allow_failure), 5 assertion-pack agents
(response_time_ms, ssl_cert, redis_ping + memcached_version, grpc_health, prometheus).
5 passed first review, 1 needed hardening (process_running), 1 had minor fixes.

**Key constraint**: tasks that required "check if X exists" or "reference existing pattern"
were pre-analyzed by Opus before dispatch — GLM agents received exact file paths and
insertion points, not discovery tasks.
