#### Format Rules

**Subject**: `type(scope): imperative summary` ‚Äî max 72 chars, lowercase after colon.
**Body**: Blank line after subject. Imperative bullet points: "Add X", "Fix Y", "Remove Z" ‚Äî never paragraphs.
**Footer**: `Refs: FEAT-xxx` when CCS detected issue refs. `BREAKING CHANGE:` when applicable.

#### Mandatory Rules

- **Never** produce a single-line message for `feat` commits touching 3+ files ‚Äî the body is required.
- **Always** include the WHY for features: what does this enable? What problem does it solve?
- **Always** include test counts when tests are added: "21 tests covering..." not just "add tests".
- **Always** reference issues/roadmap items from CCS analysis in footer: `Refs: FEAT-xxx, ROAD-yyy`.
- **No AI attribution** ‚Äî no co-authored-by, no emoji signatures, no "Generated with" lines.
- **No prose paragraphs** ‚Äî bullets only. Each bullet starts with an imperative verb.
